## "A Succulent Chinese Meal Release of Working WebSDR and KiwiSDR"

The headline is the WebSDR / KiwiSDR remote-receive layer, which had been quietly broken for everyone for a while and is now genuinely good. **Listen through any of 21 curated public receivers around the world while transmitting on your local rig** — useful when you're chasing weak POTA spots through local QRM, or when band conditions on your antenna look nothing like what's actually on the air. Tune from POTACAT, the receiver follows.

## KiwiSDR — actually plays audio now

Several stacked bugs had reduced the KiwiSDR path to "loud homogeneous noise that doesn't change between bands." All fixed:

- **Audio decoder was reading bytes in the wrong endianness.** Uncompressed PCM from a KiwiSDR is big-endian int16; we were reading little-endian, which byte-swaps every sample and turns smooth audio into uniformly-distributed noise across the full int16 range. This was the actual root cause behind every "loud noise" / "doesn't change between bands" report. Switched to `readInt16BE`, real audio appeared.
- **Frequency unit was wrong.** `SET freq=` is in kHz, not MHz; we were dividing kHz by 1000 before sending, so 14074 kHz arrived at the receiver as 14 kHz (VLF). Receiver tuned somewhere with no signal, AGC pumped the noise floor to full scale, and audio came out as the loud blast people kept reporting. Fixed.
- **`SND` header offsets were off by 3 bytes** — every audio sample was reading from misaligned bytes. S-meter values now decode to realistic dBm (`0.1*raw - 127`); audio starts at the correct byte 10.
- **`SET AR OK` now matches what kiwiclient.py does** — `in=<rate> out=<rate>` symmetric at the rate the server reports (12 kHz on most kiwis, 20.25 kHz on the kiwisdr.com proxy nodes). Renderer plays back at the rate the server actually sent.
- **Auth: send `SET ident_user=<callsign>`** after auth — many public KiwiSDRs (Bucks, others) reject connections without a callsign with `badp=5` and drop the socket within 50 ms. Also send `SET override_inactivity_timeout=1` (matches kiwiclient.py). For sysops who gate extended-listener time on callsign-as-password, use the operator's callsign as the auth password when no explicit password is set.
- **`too_busy=0` is a heartbeat, not a kick.** Treating any `too_busy` keyword as fatal made every session disconnect after the first heartbeat (~10 s in). Now we parse the value and only disconnect on non-zero. This was the actual culprit behind "kiwi keeps disconnecting me" reports — the workarounds we tried first (callsign-as-password, override_inactivity_timeout) are still useful for tlimit-restricted kiwis but weren't the proximate cause.
- **No-audio watchdog.** Some KiwiSDRs accept the connection but never send `audio_init` because the sysop reserves channels for password-holders and you're an anonymous guest. We now detect that and surface a clear error after 8 s — "KiwiSDR connected but no audio received within 8s — server may be full" — instead of silently waiting forever.

If you turn on the verbose CAT log, the first 12 server messages from every KiwiSDR connection are logged unfiltered now, plus per-frame audio diagnostics (sample range, mean amplitude, s-meter) for the first three audio packets. Future audio bugs should be diagnosable without a packet capture.

## WebSDR.org — new, working

There's been a `lib/websdr.js` shim in the codebase for a while, but it never matched PA3FWM's actual protocol — it tried raw HTTP/1.0 against a guessed URL and made wrong assumptions about audio encoding. Replaced with a real client.

- **Real WebSocket transport.** Connects to `ws://<host>:<port>/~~stream` with the `Origin` header the server requires (this single missing header was why most attempts failed-to-connect-cleanly).
- **Both audio codecs supported.** The PA3FWM protocol is a byte-tagged stream that can switch between µ-law and a custom 20-tap-LMS-predictor ADPCM mid-session; both decoders are now implemented and tested.
- **Tune without reconnect.** QSY just sends `GET /~~param?f=…&band=…&mode=…` over the existing socket. Sub-second response times.
- **Per-site band auto-discovery.** Each WebSDR.org receiver has its own band layout (Twente is one big HF band 0–29 MHz; kfsdr is per-ham-band; na5b.com is VHF-only). On connect we fetch `/tmp/bandinfo.js` from the receiver and pick the right `band=` index for whatever frequency you're tuning. If you ask a VHF-only receiver for an HF frequency, the verbose log emits a clear warning naming the available bands.
- **Sample-rate handling is dynamic.** WebSDR.org sites advertise different rates (7119, 8000, 11025, 14238, 16000 Hz observed) and can switch mid-stream. The renderer plays at the source rate; the browser handles output resampling.
- **No tlimit, no callsign requirement.** Unlike KiwiSDR, the PA3FWM software has no built-in time limit and no auth. We send `name=POTACAT_<callsign>` in the URL so the sysop sees who's listening, but it's not gated.

Verified working against three independent sites (Twente NL, KFSDR Kansas, NA5B Maryland) with a live-network test in the new test suite. 28 offline tests cover the µ-law LUT, frame parser, ADPCM determinism, band picker, and reconnection behavior.

## Settings → WebSDR — Browse station directory

Each of the three WebSDR slots in Settings has a new **Browse…** button that opens a curated list of 21 publicly-accessible receivers, every one probed live before the release went out. Filter by region (USA / Europe / Asia-Pacific) or coverage (HF / VHF), search by name or location, click **Use →** to fill the slot. Manual entry still works — the picker is just a quick-pick over the existing fields.

Three stations are flagged **Recommended** (institutional, multi-year uptime):

- **Twente** (Netherlands) — university-operated since 2008, full HF
- **KPH Coast** (Point Reyes CA) — Maritime Radio Historical Society at the historic KPH coastal station, full HF
- **Bucks PA** — local-ish East Coast US, full HF

Live directory links (rx.linkfanel.net for KiwiSDR, websdr.org for WebSDR.org) sit below the slots so you can hunt for stations not in the curated list.

A new `scripts/probe-sdrs.js` rechecks every entry's status endpoint with a 5-second timeout — that's how the v1.5.13 list was assembled and is the recommended pattern for refreshing it.

## Mac builds (continuing)

Both Apple Silicon and Intel macOS builds shipped in v1.5.12 are unchanged. Intel users still on `:8073` for KiwiSDR should now see all the audio fixes above; the protocol layer is identical across Mac architectures.
