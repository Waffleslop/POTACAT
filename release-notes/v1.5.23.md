# Spot Modes, QRZ Logins, and a Hotkey Trace

A small fixes release: POTA spots with a blank mode now get one, QRZ login failures from a `%` password explain themselves, and the configurable-hotkey path logs what it sees.

## POTA spots no longer show a blank mode

The POTA API sometimes sends a spot with no `mode` field — the spotter omitted it, or it came from an auto-spotting source. POTACAT passed that straight through, so the spot landed in your list with an empty Mode column (and slipped past mode filters).

Now, when the mode is missing, POTACAT reads it back out of the spot's comment text. Most of those "blank" spots actually say `CW`, `SSB`, `FT8`, etc. right in the comment — so they get the right mode, show up in the right filter, and tune with the right mode. Only fires when the API mode is genuinely empty; a real mode from the API is never overridden.

## QRZ login failures explain a `%` in your password

QRZ's servers currently mishandle passwords that contain a `%` character — it also breaks QRZ's own password-change page. POTACAT sends the password correctly, so there's nothing to fix on this end, but the login just failed with a generic error.

Now, if your QRZ password has a `%` in it and login fails, the error says so directly and tells you that changing your QRZ password resolves it.

## Configurable hotkeys now log what they see

A report of `Ctrl+F`-key hotkeys not firing couldn't be pinned down because the hotkey path was completely silent — nothing in the log either way. The keydown handler now logs each key it sees, whether it matched a binding, and which action ran. If a custom hotkey isn't working for you, open the dev console (View → Toggle Developer Tools) and the `[Hotkey]` lines will show exactly where it's getting lost.

---

**Tailscale users on macOS:** if Settings → ECHOCAT says "Tailscale not detected" but you have it installed, enable MagicDNS in your [admin DNS console](https://login.tailscale.com/admin/dns). The status line tells you the exact toggle and links straight to the page.

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```
