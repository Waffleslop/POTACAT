A bug-fix and polish release driven by the last week's user reports — a stack of FT-710 CW work, a fix for the POTA.app sign-in window closing too early, cross-source spot deduplication, and a new PC-side TX audio meter for diagnosing low-audio-out issues.

## FT-710 CW Keying

Multiple FT-710 owners hit a chain of issues with CW text-send. All addressed:

- **`KY` command format fix.** POTACAT was sending `KY0 AGN AGN PSE;` — Yaesu's documented format is `KY <text>;` where the byte right after `KY` is a *space* (P1 = 0x20), not a digit. The radio's parser silently dropped the malformed frame.
- **CAT-1 timeout note on connect.** The radio's factory default of 10 ms is too short and produces spurious `?;` rejections on legitimate commands. POTACAT now logs a hint pointing at the menu setting (`OPERATION SETTING → GENERAL → CAT-1 TIME OUT TIMER → 1000 ms`).
- **ATU button hidden on the FT-710.** The documented `AC002;` (ATU Tuning Start) is rejected by Yaesu firmware on this radio, and the `AC011;` POTACAT was falling back to is accepted but never starts a tune cycle. Front-panel `TUNE` button only — clicking a button that did nothing was worse than no button.
- **CW text-send via DTR keying.** The FT-710's internal CW keyer doesn't reliably play out KY-buffered text even with the format fix. POTACAT now generates morse locally and pulses DTR on the configured CW Key Port — same effect as a hand-key on the radio's CW jack, so it bypasses the firmware bug entirely. Setup: `Settings → Network → CW Key Port = /dev/ttyUSB1` (or your CAT-2 device), and on the radio set `OPERATION SETTING → CAT/LINEAR/TUNER → USB Keying (CW) = DTR`.
- **Linux `cdc_acm` fallback.** On some Linux kernels the `cdc_acm` driver rejects `TIOCMSET` (the ioctl node-serialport uses) but accepts `TIOCMBIS`/`TIOCMBIC` (the ioctl pyserial uses). When that's detected POTACAT now shells out to `python3 + pyserial` for CW keying instead of failing silently. Requires `pyserial` (`pip install pyserial` or your distro's `python3-pyserial` package).
- **Aggressive DTR/RTS pin-drop on key port open** with logging at three checkpoints, plus `hupcl:true` so the OS lowers DTR on close even if the explicit drop fails. Several reporters had the radio stuck in continuous TX after POTACAT exited; that's gone.

## POTA.app Sign-In

- **Sign-in window no longer auto-closes mid-typing.** POTA.app uses AWS Cognito for SSO, so after clicking "Sign in" the URL redirects to `parksontheair.auth.us-east-2.amazoncognito.com` with `pota.app` URL-encoded in the `redirect_uri` query param. The earlier auto-close did a substring match on `pota.app` and tore the window down before the user could enter credentials. Now we parse the URL and only auto-close when the actual hostname is `pota.app` and we're off the login route.
- **Auto-close + clearer error on first connect.** When the user signs in successfully, POTACAT detects the navigation back to pota.app and closes the window automatically (the sign-in window used to hang waiting for manual close). And if the JWT can't be captured from localStorage, the connect fails cleanly with a specific message instead of silently 403'ing on the first sync.

## Spots

- **Cross-source dedup.** When the same activator + frequency is reported by POTA, SOTA, LLOTA, WWFF, CW Spots, and DX cluster simultaneously, the table now shows **one row** with all the matching source badges instead of three separate rows. Color and bandspread render follow the highest-priority source: POTA > SOTA > LLOTA > WWFF > CW > DX. RBN / PSKReporter / FreeDV / Net pass through unchanged.
- **Spot history (ⓘ) icon moved to the Reference column** instead of next to the callsign. Per user feedback, that placement reads more naturally.
- **POTA / WWFF spot history is now accumulated locally** instead of relying on `pota.app/spot/comments` — that endpoint returns empty for almost everyone, which is why "No prior spots found" was always shown for POTA. POTACAT now snapshots raw spots from the standard polls (deduped by `spotId`) and the popover shows real history within a few minutes of launch. The popover also surfaces pota.app's total activation spot count so users see upstream context even when local history is sparse.

## Bandspread

- **Slaved to Table View.** The bandspread popout now mirrors Table View's filters and band — same source toggles, watchlist, age limits, and mode filters apply. The band picker dropdown is gone; the strip auto-follows the band Table View is on (or the band of the current radio frequency when Table is showing multiple bands).
- **VFO cursor.** A 2px red line spans the full strip with a freq+mode pennant ("14.032 CW") at the top so you can see exactly where the radio is tuned. The spot you're parked on is drawn in a high-contrast accent with a bold label.
- **Click-to-tune syncs with Table View.** Clicking a spot in the bandspread now highlights and scrolls the matching row in Table View, same as clicking the freq cell directly.
- **Zoom buttons** in the toolbar (`+` / `−` / Reset) for users without a scroll wheel. Anchored at the current view's center, same math as wheel zoom.

## TX Audio Meter

New peak meter for diagnosing low-audio TX:

- **Desktop VFO popout** — fourth meter row labeled `TX` next to S/SWR/ALC. Shows PC-side peak from voice macros, PTT-Mic, and ECHOCAT phone audio (the ECHOCAT path used to be a hidden audio bridge with no visibility). Compare against ALC: TX moves but ALC flat = raise USB MOD LEVEL on the radio. Both flat = check Windows audio output device.
- **Phone (ECHOCAT)** — TX bar in the meter strip and a "TX Level" row in `Settings → Audio`, both driven by the same desktop measurements over WebSocket.

## ECHOCAT

- **Voice macros refused in CW mode.** Clicking a voice macro in CW would key the radio (TX light on) but no audio reached the air. Now you get a clear alert: "Voice macros need a voice-capable mode (USB/LSB/AM/FM or DATA/DIGI). The radio is currently in CW." Plus a confirm-with-warning when no audio output device is configured.
- **CW paddle sidetone clipping** on phones (KQ3Q's report: each element was being truncated by the start of the next) — the paddle keyer was using the same unreliable `setTimeout` pattern the text macro path explicitly avoids. Switched to Web Audio scheduling with anchored gain ramps.
- **Phone-logged QSOs now include QRZ data** (operator name, state, county, gridsquare, country) before forwarding to N3FJP / ADIF. Previously the desktop banner logger enriched these fields but the ECHOCAT receive path didn't, so phone-logged contacts landed in N3FJP with the operator info blank. Park location overrides home QTH for POTA P2P contacts, matching desktop behavior.

## Other

- **"What's New" dialog reliability.** The version-seen marker now writes to localStorage in addition to settings.json, with the synchronous write happening *before* the async IPC. Caused by races on slow disks / forced exits, the dialog was re-appearing on every launch for some users.
- **Settings dialog stability on tiling Wayland.** Added `scrollbar-gutter: stable` so scrollbar appearing/disappearing on async content (audio device enumeration, rig editor expand) can't trigger the layout shift Hyprland was animating as the entire window sliding left.
- **Cross-source spot dedup mentioned in Spots above also affects bandspread color** — POTA-priority means a spot reported by both POTA and DX cluster shows in green (POTA) on the bandspread, not purple (DXC).

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
