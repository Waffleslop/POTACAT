**Hotfix for 1.9.0/1.9.1.** If you pair a phone to the desktop, please update.

## 📱 Pairing a new device now works (for real)

The desktop **"Pair request" approval window was shipping without its support script** in the installer — so it came up blank (no device name or fingerprint) and the **Approve button did nothing**, which left the phone's request to time out as *denied*. The fix in 1.9.1 addressed a related timing issue but not this one, so pairing was still broken in the packaged app. 1.9.2 bundles the missing file (and hardens the packaging so every popup window's script is always included), so the pairing window shows the device details and Approve/Deny work.

This only affected the downloadable installers — running from source was never impacted.
