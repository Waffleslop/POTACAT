A focused follow-up to v1.5.4 — three bug fixes from yesterday's user reports.

## POTA.app Sign-In

- **JWT capture rewrite.** v1.5.4 fixed the auto-close (Cognito SSO redirect was mistaken for a successful login), but the post-login token harvest was still failing for some users with "No JWT found in pota.app localStorage." Cause: pota.app uses AWS Amplify v6, which defaults to **memory-only token storage** — the JWT exists in the SPA's JS heap but never lands in `localStorage` or `sessionStorage`, so any DOM-storage scan misses it.

  Three-pronged capture now:
  1. **Authorization-header interception** (new): a `webRequest` filter on the auth window's session captures the Bearer token from the SPA's own outbound `api.pota.app` API calls. The SPA always sends the JWT explicitly to its own API — that's the only way the backend can authenticate it — so the token is observable on the wire regardless of where Amplify stashed it internally. This is the most reliable path and works for memory-only storage.
  2. **Live-window storage scan** (new): if the header isn't captured, scan the still-alive login window's `localStorage` *and* `sessionStorage` 2.5 s after auto-close detects login (was 1.5 s in v1.5.4 and only `localStorage`).
  3. **Hidden-window fallback** (existing): for the rare case the user closes the window manually before the live harvest runs.

## Rig Control

- **FT-857 mode changes via rigctld now follow band changes again.** v1.5.4 started sending mode-appropriate passbands (`M USB 2400`, `M PKTUSB 3000`, etc.) along with mode commands to fix an FT-991 issue where FT8 was waking up at 500 Hz. Problem is, the FT-857 only has fixed narrow/wide filters — when hamlib sees `M USB 2400` for a rig that can't accept arbitrary passbands, it rejects the whole command with `RPRT -1` and the mode change is silently dropped. POTACAT now only sends an explicit passband when the rig declares filter support; rigs without it (FT-857, FT-817/818) get `M <mode> 0` (backend default) which is universally accepted. (AG5B report.)

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
