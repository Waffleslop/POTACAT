Another step in the CW-over-a-key-port fix: the first CW send of a session now keys the radio instead of quietly using an unreliable path.

## First CW send of a session now keys through the CW Key Port

On rigs that key CW through a dedicated CW Key Port (such as the FT-891 and FT-710), the very first send after launch could produce sidetone but no actual transmit. The CW Key Port opens in the background, and the first send went out a moment before that finished — so it fell back to the radio's CAT keying command, which on these rigs puts the radio into transmit with no Morse.

POTACAT now treats a configured CW Key Port as the one and only route for CW text on these rigs. The first send waits the brief moment for the port to be ready rather than falling back, so it keys the radio the same way every later send does. If the port genuinely can't be opened, POTACAT now says so clearly instead of silently keying the wrong way.
