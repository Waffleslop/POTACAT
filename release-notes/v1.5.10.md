A small bug-fix follow-up to v1.5.9, addressing two PTT-related reports.

## SSB-over-DATA only fires when it makes sense

KL7AC (TS-890S) and KK9AWA (IC-7300) both reported the same symptom: keying PTT in USB switched the rig to PKTUSB / USB-D, disabling the rig's own mic and producing silent transmission. Hand-mic operators using POTACAT for tuning + PTT were getting their mic muted on every keypress.

The setting still does what it's designed for — keep the desktop's local mic from bleeding into the TX when audio is coming from ECHOCAT phone audio, voice macros, or FreeDV — but it now only fires when audio is actually flowing through the rig's USB CODEC. Specifically, the auto-switch kicks in only when:

- ECHOCAT audio bridge is alive (user has tapped **Tap to Connect Audio** on phone), OR
- Voice macro is currently playing, OR
- FreeDV engine is running.

For naked PTT (mic-in-hand or hardware footswitch), the mode is left alone and the rig's own mic transmits as expected. Anyone using the audio paths the setting was designed for still gets the same mic-bleed protection.

Help text reworded in Settings → Audio so the gating is explicit.

## DigiRig PTT-Port misconfiguration caught at save

N4RDX on v1.5.9 set both his CAT serial port and his PTT Port to COM4 — which is the symptom we'd designed v1.5.8's PTT Port field to fix in the first place. DigiRig presents two virtual COM ports — one for CI-V data, one wired to the rig's PTT input via DTR — and they have to be different. Setting both to the same port silently broke TX.

Three guards added:

1. **Save-time alert** in the rig editor: trying to save with `pttPort === serialPort` now pops a clear modal explaining the two-port nature of DigiRig and where to find the second COM port (Device Manager → Ports). Refuses to save until fixed.
2. **Spawn-time guard** in `spawnRigctld`: if the same misconfiguration somehow lands on disk anyway (legacy settings file, manual JSON edit), the spawn logs an explicit `[rigctld] CONFIG ERROR` line and skips the `--ptt-type=DTR` args instead of passing them to the wrong port.
3. **Clearer help text** under the PTT Port field — bold call-out that PTT and CAT must be on different COM ports, with a Device Manager hint.

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
