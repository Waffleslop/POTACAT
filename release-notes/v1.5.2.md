Biggest batch of user-reported bug fixes since launch, a stack of ECHOCAT upgrades, and a pile of Icom CI-V improvements — including a brand-new CI-V over TCP transport. Special shout-out to **KM4CFT** for a heroic reporting streak on Linux / cdc-acm / CW paddle corner cases.

## Icom / CI-V

- **CI-V over TCP** — new `civ-tcp` CAT target type. Talk to Icom radios via a networked CI-V server (including Raspberry Pi CI-V bridges, OpenWebRX, and RS-BA1-style network gateways) without needing a local serial port.
- **IC-7200** added to the Icom / CI-V rig list.
- **Force DTR/RTS low on Icom connect** so USB(A) DTR CW-keyed rigs don't immediately key the radio the moment you switch the computer over to CW. Reported by KM4CFT.
- **CW on Icom without a dedicated key port** — paddle now toggles the main CAT port's DTR when no separate key port is configured, so one-cable Icom setups don't need extra hardware.
- **Log every Icom 0x17 CW frame** in the verbose CAT log for diagnostic work.

## CW Paddle + rigctld Hardening

This release takes the "CW silently does the wrong thing" class of bugs out back.

- **Refuse to PTT instead of silent TX** — if the CW key port's DTR/RTS lines can't actually be asserted (common on some Linux cdc-acm drivers), POTACAT now refuses to key the radio and surfaces a clear error, rather than silently failing mid-CQ. Reported by KM4CFT.
- **Latch + explain DTR pin errors** so paddle keying fails cleanly with a specific "this pin couldn't be set" message instead of a generic failure.
- **Refuse to open the same COM port as CAT serial** for the CW key — catches a whole class of "why does CAT keep dropping" issues.
- **Shorter break-in + ghost-press latch debounce** — cleaner behavior on fast paddle sends, fewer false triggers from mechanical bounce. Reported by KM4CFT.
- **Yaesu KY text macros** are now wrapped in an explicit `TX1 ... ; TX0` with a delayed release, so text-mode CW macros actually key and unkey reliably.
- **rigctld mode-appropriate default passband** — stopped sending `0` (backend default) when changing modes; rigctld now gets the correct default bandwidth for the mode it's switching to.
- **Log DTR/RTS deassert ACK** for Linux cdc-acm diagnosis. Reported by KM4CFT.

## ECHOCAT

- **Full VFO View with opt-in widgets** — meters, rig control, filter, CW keyer, voice PTT, and custom CAT commands are each toggleable. Build the VFO surface that matches how you operate.
- **iPad support: split-view, VFO lock, aligned spots, HF Net info, Hide button.** Working from an iPad on the couch is now a first-class experience.
- **Map popout + rig On/Off** in the ECHOCAT toolbar. Plus Bandspread, CW XIT legacy support, 6-fer support, and QRZ dedup in the operator flow.
- **Audio UX** — docs and in-app hints now explain the per-mode USB routing quirk (voice vs DATA sound-card routing) that has tripped up first-time ECHOCAT users.
- **CW stuck-key defense** — if the phone loses focus mid-key, ECHOCAT actively releases the paddle on the rig side.
- **"TX with no RF" hint** — if you key up but no RF is actually coming out, ECHOCAT now surfaces the most common causes (split, attenuator, wrong antenna) instead of leaving you staring at a dead SWR meter.

## Spots, Filters, Logging

- **Worked-today match now includes the park ref** — a roving activator who changes parks during the day no longer gets grayed out from your first QSO of the day.
- **POTA.app sync (phase 1)** — initial scaffolding for syncing logged POTA contacts up to POTA.app. (Activation side will follow.)
- **Extra UDP broadcast is now a persistent heartbeat bridge** — survives disconnects and sends a keepalive so downstream loggers (N1MM, Log4OM, etc.) know POTACAT is still up.
- **Banner logger stops clobbering user-typed RST** with the mode-based default — once you've typed `57`, it stays `57`.
- **Spot history popover** ⓘ icon on POTA/WWFF/DX Cluster/RBN rows — quick access to what you've heard from this station/park before.
- **DX cluster runaway reconnects** when feeds are disabled are now throttled. New cluster: **S50CLX**.
- **Scan dwells once per frequency per pass** — the adjacent-duplicate skip was too greedy on busy bands.

## Other Rigs

- **TS-2000 rig profile** (no `RM1` command — avoids the SWR-read stall). Reported by K4VL.
- **ALC meter + Yaesu RM channel fix + TX-gated meter polling** — meter reads are now only triggered while TX is keyed on the rigs that need it, ending a long-standing idle-CAT-chatter complaint.
- **Linux launcher** alongside the existing Mac/Windows variants.

## QRZ + SSTV

- **QRZ Logbook API** responses with HTML-encoded ADIF payloads now decode correctly — some contacts that were silently dropped are now imported.
- **SSTV Scottie S2 added**, S1 color fix, SSTV gallery auto-refresh, and macro dedup so the same SSTV macro doesn't fire twice.
- **JTCAT spot highlight** when the matching spot is live in your spot list.

## Fixes

- **Fix startup lag** + massive Hide column for existing users (the Hide column was unbounded in width from earlier migrations).
- **S-meter show toggle** actually toggles now, and the **QSO duplicate explosion** reported in-app has been traced and squashed.
- **Null-safe view-menu button wiring** — stuck app on launch for KM4CFT after a view-menu cleanup that assumed every button existed.
- **Rig-list overflow on long serial paths** (`/dev/serial/by-id/...`) — Edit/Delete buttons stay visible. Reported by KM4CFT.
- **Linux serial port display** now shows `/dev/serial/by-id/` aliases instead of opaque `/dev/ttyUSB0` paths — pick the right radio the first time. Reported by KM4CFT.

## Scope Note

- **HamDRM digital SSTV port was reverted.** After weeks 2–3 of experimental work it was clear the investment was pulling focus off POTA-first priorities. The interop test kit and Raspberry Pi setup notes remain for anyone who wants to continue the experiment locally.

---

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
