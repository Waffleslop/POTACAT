A follow-up to the last release: CW text now sends all the way through instead of stopping partway.

## CW text-send no longer cuts itself off

With CW keying working again through a CW Key Port, sending a longer message — a full CQ, for instance — could start and then stop a few seconds in. The cause was a duplicate send: the same macro arriving twice a few seconds apart, where each new send cancels the one already in progress, so the first transmission got chopped.

POTACAT now ignores a repeated identical CW send while the previous one is still going, so the message finishes cleanly. Sending different text still interrupts and takes over as before, and the Stop button still aborts immediately.
