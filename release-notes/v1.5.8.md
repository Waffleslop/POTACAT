A bug-fix follow-up to v1.5.7. Headline is the ECHOCAT renderer regression — a single missing reference broke the spots table for everyone on v1.5.7 — plus a stack of user-reported polish.

## ECHOCAT — Critical fix

- **Spots table not loading on v1.5.7.** A `window.__vfRenderAll = renderAll;` line added with the v1.5.7 beam-heading toggle was placed inside the wrong IIFE — `renderAll` is defined inside `setupVfoFullview`, but the assignment lived inside the *next* IIFE `setupVfoWidgets` where that name is out of scope. Strict mode threw a `ReferenceError: renderAll is not defined`, the throw halted every later setup block, and the page rendered only the static HTML — VFO knob and the "Tap to Connect Audio" button, nothing else. (KK4DF, KM4CFT, and others.)
- **Diagnostics for the next failure.** ECHOCAT now emits server lifecycle events into the desktop's Verbose log (`Server listening on https://0.0.0.0:7300`, `New connection from 192.168.x.y`, `Failed to serve / : <reason>`, etc.) instead of writing them only to stdout where installed users can't see them. Plus a top-level error trap in the phone client paints any uncaught JS error onto the page in a red banner — no more silent partial renders.
- **`/health` endpoint** returns plain `ok` — gives users a way to prove the server is reachable from their phone even when the SPA itself errors.

## POTA.app Sync

- **403 errors that survived the v1.5.7 refresh-token fix.** KD2TJU's log showed a freshly-refreshed token still 403'ing on `/user/stats/parks/csv` — meaning the API authorizer was rejecting on request *shape*, not token freshness. Browser fetches from pota.app always carry `Origin`, `Referer`, and `Sec-Fetch-*` headers; raw `net.request` calls from POTACAT didn't, and the gateway was gating on them. Now sent on every API call.

## ECHOCAT (Phone)

- **FreeDV no longer hides the PTT button.** v1.5.7's FreeDV mode override sets the rig to PKTLSB / PKTUSB (data sub-bands), and ECHOCAT's voice-mode detector didn't recognize those as voice-capable carriers — so it hid PTT. PKTUSB / PKTLSB / DIGU / DIGL / USB-D / LSB-D now all count as voice modes. Same for the SSB macro panel visibility. (Chris G7 report.)
- **EarPods button no longer hijacks PTT.** v1.5.7 unconditionally registered Media Session play/pause/stop handlers so wired earbuds could toggle PTT — but that meant every EarPods press grabbed PTT regardless of whether the user wanted it. The "BT PTT" toggle in Settings is the explicit opt-in path; the unconditional registration has been removed. Wired-earbud users who want PTT on the click button: enable BT PTT. (Chris G7 report.)
- **FreeDV squelch persists across sessions.** Previously the slider reset to 2 dB on every restart; now `freedvSquelch` is mirrored to `settings.json` and re-applied after every `freedvEngine.start()`. (Chris G7 request.)
- **Pin the clicked spot in the table.** When you click a row to tune, that spot is now exempt from age cull, source toggles, band/mode/region filters, and hide-worked — it stays in the table until you click somewhere else. The row also re-centers in the viewport when new spots reflow it off-screen, but only when out of view (so manual scrolling isn't fought). Closes a workflow break where a weak POTA station's spot would time out from pota.app mid-QSO and force the operator to log manually. (N5SKT report.)

## Rig Control

- **PTT Port field for DigiRig / SignaLink users.** Older Icom / Yaesu rigs (IC-706MKIIG, etc.) accept a CAT PTT command but don't reroute audio to the USB CODEC — so POTACAT's rigctld setup keyed the radio but no audio reached the air, while WSJT-X (which drives DigiRig's DTR-keyed PTT directly) worked fine. Settings → Rig → Hamlib config now has an optional PTT Port dropdown; pick your interface's PTT-side serial port and rigctld will be spawned with `--ptt-type=DTR --ptt-file=<port>` so PTT and audio gating match. Empty = unchanged behavior. (N4RDX report.)
- **rigctld errors now say what they mean.** When the rig backend returns an `RPRT -N` code, POTACAT translates it before showing — `RPRT -5` becomes "Communication timed out — radio not responding (check cable / baud rate / power)", `RPRT -20` becomes "Radio reports it is powered off — turn the radio on (or use POTACAT Rig → Power On) and try again", and so on for the full hamlib error table. Surfaces in both the Test Connection dialog and the live CAT log. (AB9AI report.)
- **IC-706MKIIG added to the Icom dropdown.** Conservative caps — basic freq/mode tuning works; the rig predates the CI-V level/function sub-commands so filter switching, ATU, NB control, and remote power-on aren't wired up. (N4RDX request.)

## Linux Build

- **Hamlib 4.7.1 compiled from source.** Ubuntu-latest's `libhamlib4` apt package is 4.5.5, which predates FTX-1 support (added in hamlib 4.6). The Linux runner now downloads, builds, and bundles hamlib 4.7.1 — so the AppImage / deb / rpm all carry a current rigctld with the FTX-1, IC-7300 MK II, and other recent rig backends. A build-time guard fails the release if FTX-1 falls out of `rigctl --list`, so this can't silently regress. (royw3888 report.)

## Cloud Sync

- **Readable error messages on server hiccups.** When `api.potacat.com` briefly returns 502/503/504 (gateway issues), the Verify Subscription / Sync Now dialogs no longer dump the raw nginx HTML body. You get a single line: "POTACAT cloud is temporarily unavailable (HTTP 503). Please try again in a few minutes — if it keeps happening, email casey@potacat.com." Cached subscription status is preserved across transient server hiccups, so a 503 won't downgrade an active supporter to "trial" in the UI.

## Other

- **Update banner duplicate version text.** The "POTACAT update available" banner now reads cleanly when the GitHub release title is just `v1.5.8` instead of "v1.5.8: v1.5.8". (bjh report.)
- **Show S-Meter / SWR checkbox stays unchecked across restarts.** v1.5.5 fixed the persistence by mirroring to `settings.json`, but the boot-time initialization still read only localStorage, which Electron can reset across installer updates. Now reconciled at startup so the box stays hidden if you turned it off — without needing to open Settings to re-trigger the fix. (bjh report on v1.5.7.)

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
