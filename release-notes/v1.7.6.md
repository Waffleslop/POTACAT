# The Version to Align with iOS 1.0.0 Release

**Headline:** Coordinated release alongside the ECHOCAT iOS 1.0.0 launch. The desktop ships a community DXpedition feed, much richer DXP metadata in the spot table, per-source DXP visibility toggles, a right-click "add to watchlist" gesture, FTX-1 Field + Optima support, and a fix for JTCAT click-to-reply that was a deal-breaker for users hunting non-CQ stations.

## Community DXpedition Feed (new)

POTACAT now reads from a community-aggregated DXpedition feed hosted at `dxpeditions.potacat.com`. The aggregator scrapes **DX-World**, **DXNews**, and **NG3K ADXO** every 6 hours server-side, normalizes them into a canonical schema, and serves the union. Currently ~108 active records vs. the handful the old danplanet iCal source was producing.

- **Cross-source corroboration** — when the same call appears in multiple feeds, the badge tooltip shows the source list (e.g. *"Sources (3): dx-world, dxnews, ng3k"*). Three feeds reporting the same op is strong evidence it's a real upcoming operation, not a typo.
- **Per-source visibility toggles** — under the master "DX Expeditions" row in the Spots dropdown, four indented children let you toggle Club Log, DX-World, DXNews, and NG3K independently. Trust the sources you want, suppress the ones you don't.
- **DXpedition metadata in the Notes column** — DXP-flagged rows now show structured info inline:
  ```
  DXP: Tanzania · Jun 8-Jul 7, 2026 · By NK8O · 40-6m CW FT8 FT4 · QSL LoTW
  ```
  The original POTA / cluster comment stays behind a `|` separator.
- **Hover the DXP badge** for the full RSS description plus the contributing-feed list. Multi-line tooltip — no popover dance.
- **Cadence** — desktop pulls every 6 hours from the worker. The worker handles the actual scraping, retries, and per-source isolation (one source going down doesn't kill the others).

The old danplanet iCal source has been retired. Club Log's "ops uploading logs in the last 7 days" feed is still on — it complements the announced lists with "this op is on the air *right now*" signal.

## Right-click a Spot → Add to Watchlist

The existing row context menu (hide spot) now also offers **Add to Watchlist** plus one button per named watchlist group. Each button toggles — click again to remove. Group buttons get a colored left edge so you can tell which group is which at a glance. No more bouncing into Settings to add a call.

## FTX-1 Field + FTX-1 Optima Support

First-class CAT support for the Yaesu FTX-1 family. New rig models with the full FTX-1 caps surface, ~15 new CAT commands (VOX gain, AGC mode, NB/DNR levels, compressor on/off + level, mic gain, monitor on/off + level, CLAR RX/TX/offset, break-in + delay with the Yaesu coded-value mapping, preamp targeting for HF/50 / VHF / UHF). The rig popover grows the matching controls — capability-gated, so existing radios show nothing new.

Optima uses the documented `AC103;` ATU sequence (per W9JL's 2026-04 confirmation). Field uses the standard one.

## JTCAT Click-to-Reply: Tail-End Anyone

**Bug:** double-clicking an FT8 decode only initiated a QSO if the station was calling CQ. Decodes like `K3SBP NA7C -12` (NA7C mid-QSO with someone else) or `WB4ABC NA7C 73` (NA7C finishing a QSO) were non-clickable. NA7C reported this as a deal-breaker.

Fix matches WSJT-X behavior: clicking any decode targets the **sender** (the right-hand callsign) when neither party is your call. Bare-call shapes only — grids, signal reports, and acks (RR73 / 73 / TU) are rejected so we don't try to call `-12` as a callsign. Same logic on the in-app decode list and the JTCAT popout.

(iOS app gets the same fix in its own 1.0.0 update — wired via the existing `jtcat-reply` WebSocket message, no server-side change needed.)

## JTCAT: Reply with a Report, Not Our Grid

When someone answered our CQ with their grid (`K3SBP NA7C EM12`) and we double-clicked it, JTCAT was sending `CQ K3SBP FN20` again instead of advancing to a signal report. Now correctly sends `NA7C K3SBP -NN` — the next phase of a real QSO.

## RBN Connection Watchdog

The RBN telnet client was getting stuck in half-open states (NAT timeout, server-side drop without FIN) and silently going dead while still reporting "connected." TCP keepalive plus a 3-minute data-arrival watchdog now detect wedged sockets and trigger reconnect. Symptom this fixes: you call CQ, RBN's website shows you being heard six times, but POTACAT's Prop tab shows zero.

## Map Tune Arc: Now Visible

The QTH-to-spot arcs on the map were getting lost against busy OSM tiles and clusters of bright markers. Now drawn as a 4px black halo under a 2px colored dashed line. Clearly more legible without overpowering the markers.

## Other Improvements

- **SSTV decode** — leader-tone purity gate stops false-locks on band noise. Several users reported SSTV not decoding for ~3 weeks; this addresses the root cause (the leader was locking on noise, jamming subsequent real decodes).
- **SSTV TX on Flex Direct** — routes audio through VITA-49 dax_tx so transmit actually works when no SmartSDR is running.
- **Flex audio OOM** — fixed the ~1.7 GB / 50-minute crash. The iOS ECHOCAT connection was pumping a silent dax_tx mic stream at ~188 packets/sec regardless of TX/RX state; now gated on actual transmit.
- **Remote Launcher** — serves HTTPS with the Tailscale cert when available, so iOS can actually reach it. Also fixed a regex bug in PID parsing that was returning bogus process IDs.
