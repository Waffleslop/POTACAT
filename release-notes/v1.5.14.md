## "ECHOCAT for iOS Lands" Release

The headline is the **ECHOCAT mobile (iOS) protocol pass** — every one of the 20 gaps the mobile dev raised during the iOS-build phase is closed in this release. Mobile pulls in propagation data, plays SDR audio over WebRTC, plays voice macros remotely, and toggles auto-SSTV / FT8 Auto-Seq from the phone. If you're running ECHOCAT mobile, this is the desktop you want.

Plus a new spot source (**Tiles on the Air**), cross-source dedupe, and a cluster of fixes from the field — DA2PK's FT-710 CW, AB9AI's FTdx3000 SWR/ALC, KF8ELA's filter bypass, KD2TJU's JTCAT tune button, K4GDJ's FTDX101 mode flip, and more.

## Tiles on the Air

A new awards program, similar in shape to POTA / SOTA / WWFF / LLOTA. POTACAT now polls the Tiles on the Air API for live spots and merges them into the table and map alongside the other sources. **On for new users by default**; existing installs get a toggle in Settings → Spots.

Cross-source dedupe runs across all five program sources now — if the same activator is spotted on POTA and Tiles in the same band/mode, the table shows one row with the higher-priority program tag rather than two duplicate rows.

## ECHOCAT mobile (iOS) — gaps closed

The mobile dev at the sister project ran a full audit of every desktop→phone message path. All 20 gaps closed across two desktop passes:

- **Gap 14 — Auto-SSTV remote toggle.** Phone tap on the AUTO-SSTV banner flips `enableAutoSstv`, persists, cancels in-flight session.
- **Gap 15/16 — PSKReporter Map spots + status to mobile.** The Prop tab on mobile now sees the same data as the desktop popout. Snapshot pushed on connect with `nextPollAt` countdown.
- **Gap 17 — Auto-SSTV defers when JTCAT is decoding.** Phone-only FT8 sessions no longer get their audio yanked at the 90-min idle threshold.
- **Gap 18 — Voice macro playback from phone.** Tap a macro slot on the phone, the desktop renderer plays the clip out the rig (PTT-on, play, PTT-off via the existing voice-macro-ptt path).
- **Gap 19 — Full RBN firehose to mobile.** New `rbn-prop-spots` channel pushes the full 500-spot array; the watchlist subset stays on the existing `spots` channel for the Spots tab.
- **Gap 20a — KiwiSDR/WebSDR audio over WebRTC.** Mobile can't decode raw PCM (no Web Audio on RN). Desktop now swaps the WebRTC peer's audio source from rig audio → Kiwi PCM via `RTCRtpSender.replaceTrack()` when Kiwi connects, with no renegotiation. Phone hears SDR audio over the same voice path it already had wired for rig audio.
- **Gap 20b — KiwiSDR schemas registered.** `kiwi-tune` (mid-session QSY), `kiwi-status`, `kiwi-smeter`, `kiwi-audio`, optional `password` on `kiwi-connect`. Plus `kiwi-tune` handler in main for phone-driven QSY.
- **Gap 11/12 — JTCAT Tune toggle + Auto Seq.** Phone FT8 screen renders the WSJT-X-style 90s steady-tone Tune button + Auto Seq toggle, both wired through the protocol with state pushed on connect.
- **Gap 13 — Launcher TLS off by default.** New installs default `https: false` so iOS doesn't reject self-signed certs.
- **Gaps 1–10** (earlier desktop pass) — `freedv-enabled`, `voice-macro-labels`, `voice-macro-sync` direction, `tune` schema, `status` field catalog, `qrz-names` registration + composition, JTCAT zero-decode after SSTV idle, meter fields in `status`.

## JTCAT — TX timing, decode timing, audio capture

Three substantial JTCAT fixes that compound into "QSOs advance on the first cycle, not the second":

- **TX-complete fires on the AudioContext clock, not buffer drain.** Mobile dev nailed this one: `AudioBufferSourceNode.onended` was lagging 1–2 s past the actual end of FT8 audio on Web Audio sinks with deep output buffers (WebRTC routes, USB CODECs with large jitter buffers). PTT held that long shrinks the next slot's RX window so much that the rig's TX→RX changeover transient strands the decoder's Costas sync. Now `setTimeout` fires `finishTx()` at `durationSec + 150 ms`, synced to the audio output clock. Compresses a typical reply cycle from 2 cycles to 1, matching WSJT-X.
- **Decode fires 1.5 s BEFORE the next slot boundary, not 0.5 s after.** The old timing sometimes ran decode AFTER the next TX boundary had already passed, sending a stale message (the W4MAA QSO at 03:58:15 sent FN20 again instead of R-12 because R-12 wasn't pre-encoded until 583 ms after TX had already started). FT4 picks up the same fix.
- **Silence watchdog stops firing on healthy circular-buffer wraps.** The old heuristic flagged a wrapped buffer as "audio capture is dead" — reported by K3SBP as JTCAT randomly going dark mid-session.

Plus: SSTV teardown when JTCAT starts (so SSTV-idle handover doesn't leave the audio device locked), audio-capture diagnostics on cold-start, FT8 decoder choice (native vs WASM) surfaced in the Verbose CAT log.

## Propagation map — always on

POTACAT now collects RBN and PSKReporter Map data passively whenever a callsign is configured. No more "I called CQ but POTACAT didn't show me getting heard" — RBN telnet client and PSKReporter Map poller both auto-start on launch (resource cost is tiny: ~1–15 KB/s of regex-parsed text for RBN, one HTTPS poll every 5 minutes for PSKR). The Propagation popout gains a status header showing live connection state, cached spot counts, and the PSKReporter next-poll countdown.

The main-window RBN and PSKReporter chips now reflect actual connection state instead of a legacy enable-flag, so you can tell at a glance that propagation data is flowing.

## WebSDR.org CW — actually audible now

DA2PK on Twente was hearing his own transmission on the browser version of the WebSDR but silence in POTACAT. Root cause: WebSDR.org's CW mode is USB demodulation with a narrow audio filter — there is no client-side BFO. A carrier sitting exactly at the tuned frequency demodulates to 0 Hz audio, which the lower cutoff filters out. The browser frontend silently subtracts the operator's CW pitch from the tuned freq before sending; POTACAT didn't.

POTACAT now tunes 600 Hz below the displayed VFO when mode=CW (typical CW operating pitch) and widens the audio passband from 300–800 Hz to 200–1100 Hz so any rig pitch from 400–1000 Hz stays in band.

## Bug fixes from the field

- **DA2PK FT-710 CW (Linux, OpenBSD).** After the kernel's first ENOTTY on `cdc_acm`, the pyserial fallback target was being wiped on every reconnect, then the next text-send re-opened node-serialport and hit ENOTTY again. The synchronous caller fell through to the unreliable Yaesu KY command (carrier rises but no Morse). Two fixes: preserve the fallback target across same-port reconnects, and short-circuit straight to pyserial once we know TIOCMSET is rejected.
- **AB9AI FTdx3000 SWR/ALC.** Two issues. (1) S-meter race when poll order was freq→mode→smeter — the freq response was being parsed as an out-of-range smeter value and stranding the expectation. (2) SWR and ALC weren't moving during physical-mic TX because POTACAT only polled the TX-only meters when it had initiated keying itself; mic / footswitch / external PTT was invisible. Now polls hamlib's `t` command every cycle and updates `_transmitting` from the response, so meter polling tracks any TX trigger within ~500 ms.
- **KF8ELA SSB-only filter showed FT8 spots.** `onCatFrequency` was auto-pinning a random PSKR FT8 spot at the rig freq, bypassing all source filters. Removed the auto-pin from CAT freq updates — only user-click pins survive now.
- **K4GDJ FTDX101MP SSB-over-DATA on naked PTT.** `voice-macro-ptt` always passed `audio:true`, which on Yaesus flips USB → PKTUSB and disables the rig's mic. Naked PTT (VFO popout, hand mic) now uses a separate `naked-ptt` IPC path with `audio:false`.
- **KD2TJU JTCAT Tune button.** WSJT-X-style 90 s steady 1500 Hz tone for power / ALC tuning. Reuses the FT8 audio path; cancels at 90 s or on user toggle.
- **VK3AWA SDR TX mute.** When the user transmits, Kiwi/WebSDR audio is muted so the speaker doesn't blast the operator with their own CW sidetone or USB modulation passing through the SDR.
- **N7FQT spot popover dark in light mode.** `--bg-card` / `--bg` were never defined as CSS variables. Fixed to `--bg-secondary` / `--bg-primary`.
- **K0OTC strict ATNO.** Hunter wanted "show me parks I've never worked, even if logged this session." `workedParks` was merging credited + local; now tracks `creditedParks` separately for the strict-ATNO toggle.
- **AG5B multi-park editing.** SIG_INFO with comma-separated refs (e.g. `K-0001,K-0002`) wasn't splitting on import; now prompts to confirm the n-fer split and clones the QSO.
- **CASEY → Casey QRZ name normalization.** All-caps QRZ names now title-case at the source (`lib/qrz.js`), preserving mixed case (McDonald, MacGyver). Five call sites previously read raw.
- **W0MET silent-PTT fallback.** When the SmartSDR API isn't connected, `handleRemotePtt(true)` now falls back to the TS-2000 `TX;` / `RX;` shim over TCP CAT — slice selection is lost but at least the radio keys.
- **JTCAT silence watchdog.** Stops firing on healthy circular-buffer wraps (K3SBP) and skips during TX cycles when the rig mutes RX (every TX cycle was triggering a false alarm before).
- **Casey logbook popout.** Light-mode titlebar / borders / muted text / accent gold were all referencing CSS variables that didn't exist; fixed each. Save QSO button switched from `--accent-green` (forest in light mode) to `--accent-green-btn` (always mint) for contrast.

## ECHOCAT v1 protocol — Phase 0 foundation

Even if you don't run ECHOCAT mobile, the protocol module behind it is now formalized:

- `lib/echocat-protocol.js` declares the full message registry — direction (server→client, client→server, both), required fields, types — and validates every inbound message at the server.
- A `hello` handshake exchanges protocol version + client platform on connect; clients on incompatible versions get `PROTOCOL_VERSION_UNSUPPORTED` and a clear close reason instead of silent message drops.
- mDNS / Bonjour publication of `_potacat._tcp.local` for LAN discovery.
- QR-pair onboarding (one-time tokens, 5-min TTL, single-use) via `/api/pair`.
- Per-device tokens minted by pairing — no more shared-secret login.

`docs/echocat-protocol.md` and `docs/echocat-mobile-plan.md` document the protocol catalog and the mobile-app project plan respectively.

## UX / cosmetic

- **New app icons** — blue-gradient artwork; Windows installer/menu, macOS dock, Linux.
- **Logbook + New QSO** has type chips for DX / POTA / SOTA / WWFF / LLOTA / Tiles, with a "respot after save" checkbox checked by default for park sources.

- **VFO Profiles** widget gains inline rename via an edit button; the profile list pushes to ECHOCAT mobile.
- **Split mode** dropped the `S` hotkey (caused accidental split on a stray keypress); enable from Settings only. Status-bar SPLIT pill is clickable to disable with a confirm dialog.
- **Panadapter source filtering** — independent from spot table source filtering. You can now show POTA on the panadapter while filtering it out of the table, and vice versa.

## Diagnostics

- **Startup timing instrumentation.** `--startup-timing` flag emits per-stage durations to stdout so users diagnosing variable boot lag can see whether it's IPC, settings load, or a connect path.
- **WebSDR raw-frame dump.** Set `POTACAT_WEBSDR_DEBUG=1` and the verbose log shows the first 8 binary frames in hex — used to diagnose Twente sizzling on DA2PK's OpenBSD setup.
- **FT8 decoder choice in Verbose CAT log.** "NATIVE decode" vs "WASM decode" so users can confirm the C library is being used (10–50x faster than WASM).
- **JTCAT QSO state machine** logs every phase advance / no-advance decision with the reason — much easier to diagnose "why didn't it advance?" reports.
