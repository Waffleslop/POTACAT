This release recovers a radio that gets stranded in transmit when the USB connection resets during a CW transmission, and it explains why that happens so you can address the underlying cause.

## Automatic recovery from a stuck CW transmit

On rigs that key CW through a separate USB-serial adapter — for example an FT-891 with a dedicated key port — sending CW could occasionally leave the radio latched in transmit with no output, clearing only when the cable was physically unplugged. This was reported by N7BBQ.

The sequence behind it: keying CW puts the radio into transmit, and on some stations the transmit RF gets back into the USB cable and resets the USB device. On a radio whose single USB connection carries both the CAT control and the CW key line, that reset drops both at the same moment — while the key line is still held down. POTACAT's existing safety tries to release the key line the instant the link drops, but by then the port has already vanished, so the release cannot reach it and the line stays keyed.

Two things are now fixed:

- When the radio's USB connection comes back, POTACAT reopens the CW key port and forces the key line low again, releasing a line that was left stranded during the reset. In the earlier behavior the CAT port and voice PTT were restored on reconnect but the key port was not, so a stuck key stayed stuck.
- If the connection drops within a few seconds of keying CW, POTACAT now names the likely cause directly — the transmit is probably resetting the USB device (RF getting into the cable) — and suggests adding a ferrite choke to the USB cable, improving station grounding, or lowering power. It also tells you the key line will be released automatically when the radio returns, rather than only saying "check USB."

Two things worth knowing: the underlying cause is a station/hardware issue (RF entering the USB), so the durable fix is stopping that RF at its source; and if the adapter is physically unplugged in the middle of a transmission, software can no longer reach it to release the line.
