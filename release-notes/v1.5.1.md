## ECHOCAT Full VFO View

Biggest addition in this release — a dedicated, touch-friendly VFO view inside ECHOCAT. Open it from the status-bar VFO button and you get an operator-first layout: large frequency dial, dial detent on step changes, band names, and a rich op card for tuned spots showing the call/name, location, park ref, and a one-tap Log QSO button — all without leaving the VFO. In CW mode the PTT bar hides so the paddle interface has room to breathe.

- Status-bar frequency tap opens the VFO; tapping inside the VFO opens the numeric keypad — each gesture now does the obvious thing.
- Log QSO button sits beside the name/state/park card rather than under it, and the PTT bar no longer covers it.
- Fixes the empty-thin-box op card on tuned spots and the band-name clipping at certain widths.

## ECHOCAT Power Meter

Header strip now shows Power alongside S-Meter and SWR — glance at the whole RF picture without swapping views.

## SSTV Polish

- **Reply with PiP of their image** is now discoverable — the affordance was there but nobody was finding it.
- **Live-sync desktop SSTV compose to ECHOCAT phone** — build the image on your phone and watch the desktop mirror update in real time.
- **Auto-QSY on SSTV view open**, and templates saved from ECHOCAT flow back to the desktop.
- **USB everywhere for 40m SSTV QSY** — was incorrectly picking LSB.
- **HALT button on both desktop and phone**, plus PTT release on shutdown so a crash doesn't leave the radio keyed.
- Template load actually updates the display, iOS uses `img.decode` for template backgrounds (fixes flicker).
- Decoder events surface in the verbose CAT log with a 5-minute heartbeat so you can tell what the decoder was doing after the fact.
- Removed SSTV desktop auto-push — the model is now pull-only, which matches user expectations.
- Fixed the Decode Log "Hide" trap (no way to restore it once hidden).

## Rig Control &amp; ATU

- **RF-sense external ATU support** — works with LDG Z-100plus, MFJ, and other RF-sensing tuners that need a tune-up signal on key-down.
- **FTX-1 Optima (AC103 ATU variant)** now supported with ATU presets, logged to the CAT log for verification.
- **KenwoodCodec honors atuCmd preset** + every ATU and CW write is logged for troubleshooting.
- **Icom CW 0x17 frames** are now logged verbatim in the verbose CAT log.
- **Flex silent PTT failure fix** — when SmartSDR API isn't connected, PTT no longer fails silently.
- **SmartSDR API retry storm**: replaced the background retry spam with an actionable UI banner. Clearer help text on the SmartSDR host field.
- **rigctld CW + PTT writes** are logged so KO4XJ's elusive CW bug has something to grab on to.

## Spots &amp; Filters

- **Hide worked CALL+park combos** — strict 5-field match (callsign + ref + mode + band + date) filters out stations you already have for that park. No more working them twice.
- **Show NEW parks at top** — a sort modifier in the Spots dropdown that surfaces parks you haven't worked yet above the pack.

## Logging

- **Broadcast to extra UDP port** — a new Logging setting to mirror logged QSOs to a second UDP endpoint (on top of the existing one). Useful for running a secondary logger or contest app in parallel.
- **Banner Logger shows QRZ location** next to name, and supports drag-to-resize.

## Report a Bug, One-Click

- New **Report a Bug** button in Quick Settings — bundles a redacted system snapshot, rig config, and recent log tail into a prefilled Discord post. Grid square is excluded from the payload to keep location private.
- **Copy Report** now routes through the main-process clipboard, fixing the silent no-op on some systems.

## HF Nets Directory

- Added-badge is now a clickable **Remove** button so adding a net is a toggle, not a one-way operation.

## Build Pipeline

- Fresh GitHub Actions release pipeline builds Windows, macOS (arm64), and Linux in parallel, attaches `latest.yml`, `latest-mac.yml`, and `latest-linux.yml` for auto-update, and auto-updates the download links on [potacat.com](https://potacat.com/).
- Windows audio device enumeration no longer crashes when Windows blocks the mic permission — falls back to text-only device names.
- Intel Mac DMGs are a known follow-up: the current arm64-only build ships for Apple Silicon. The universal-DMG workflow (lipo-merged native addons + hamlib) is tracked as the next macOS investment.

## Small Stuff

- Pinned the Edit/Delete buttons in the rig list so long Mac/Linux device paths can't push them off-screen.
- Replaced Unicode `→` arrows with ASCII `->` in source to keep terminals and grep-heavy workflows happy.

---

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

**Intel Mac users:** build from source for now — a universal DMG is coming in a later release.
